## Context

系统目前存在的三处问题具体如下：
1. **六爻变爻倒置**：`HexagramDisplay` 中，对结构字符串进行了 `reverse()` 以实现从上到下渲染，导致动爻数组高亮映射时，最上面的上爻（index 0）错配成了初爻（position 1）。
2. **紫微宫位顺时针错排**：`ZiweiPage` 排盘将十二宫按顺时针方向分布，不符合传统紫微命盘逆时针旋转的宫位规律。
3. **梅花易数公历起卦**：梅花时间起卦直接使用了公历日期的纯数字求和，导致推算的主卦和动爻位置与真实的阴历干支完全脱节。

## Goals / Non-Goals

**Goals:**
- 修复 `HexagramDisplay` 的动爻高亮对应关系，确保动爻完美匹配。
- 将紫微十二宫排列方式变更为逆时针，遵循传统命盘数理。
- 引入可靠的农历转换工具，将公历起卦日期准确转换为阴历年月日时干支后再进行梅花起卦。
- 升级细节 UI 设计（如输入与展示面板的阴影、边框、过渡等细节），展现 impeccable 年轻国潮美学。

**Non-Goals:**
- 不重写六爻和梅花易数的底层核心卦象映射库，只修正其排盘和入参计算过程。
- 保持紫微斗数的简易排盘，不对复杂的流年飞星和百颗辅星进行全面星曜推算。

## Decisions

### 1. 动爻索引映射修改
在 `HexagramDisplay.tsx` 中，由于结构已倒序渲染，将 `position` 的映射从 `index + 1` 改为 `6 - index`。这样，数组中的第一个元素（对应屏幕最上方的上爻）将正确计算为 position 6，而最后一个元素（屏幕底部的初爻）计算为 position 1。

### 2. 十二宫逆时针旋转映射修改
在 `ZiweiPage.tsx` 中，将计算宫位数组的公式从顺时针 `(i - mingGongIndex + 12) % 12` 变更为逆时针 `(mingGongIndex - i + 12) % 12`。此举可保持地支方位（子、丑、寅）依然顺时针固定分布在网格四周，但十二宫位的名字（命宫、兄弟、夫妻）将逆时针进行环形排列，符合古法。

### 3. 安装并引入 `lunar-javascript` 处理农历换算
- **技术选择**：安装业界标准的 `lunar-javascript` 开源库。该库大小轻量，且能极好地适配 Vite 环境，准确计算出公历对应的农历天干地支及阴历月日数值。
- **换算实现**：
  - 梅花时间起卦时：将公历 `(year, month, day, hour)` 传入 `Lunar.fromYmdH` 方法。
  - 获取农历年份的地支序号（子=1...亥=12）作为年数，农历月数作为月数，农历日数作为日数，时辰地支序号（子=1...亥=12）作为时数。
  - 用这四个农历数求取梅花卦象和动爻，实现完全地道的传统推推。
  - 同样可以在起卦页面和结果页面清晰展示转换后的农历干支日期，提高应用专业感。

## Risks / Trade-offs

- **[Risk]** 引入外部 npm 包 `lunar-javascript` 增加构建体积。
  - **Mitigation**：`lunar-javascript` 无外部依赖，体积极小，且支持 Tree Shaking，在 Vite 打包中对整体 precache size 几乎无负面影响。
- **[Risk]** PWA Precaching 缓存更新。
  - **Mitigation**：在 `vite.config.ts` 中已配置 `registerType: 'autoUpdate'`，代码更新后会自动触发 PWA 资源重载。
